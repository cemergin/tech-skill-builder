# Assessment Rubric Reference

## Overview

The tech-tutor uses informal assessment to gauge learner level, adapt pacing, and suggest entry points. This is NOT formal testing — it's continuous, conversational calibration to keep the learner in their Zone of Proximal Development.

## Session Start Assessment

### Quick Calibration
When a learner starts a session, gauge their level with 1-2 conversational questions:

**For brand new learners:**
"Have you worked with [topic] before, or is this completely new?"
- If new: Start at Module 1, use heavy scaffolding
- If some exposure: "What have you used it for?" then suggest an entry point

**For returning learners:**
"Last time you completed [Module X]. Ready to pick up from [Module X+1], or want to review anything?"

**For learners who want to skip ahead:**
"Take a look at the Episode summary for [Module N]. If those concepts feel solid, you can jump to [Module N+1]. If anything looks unfamiliar, it's worth doing [Module N] — it'll take about [estimate] and sets up everything for later."

### Self-Assessment via episode.md
Each module's episode.md contains:
- **Key Concepts** — the learner can scan to see if they know these
- **Prerequisites** with a self-assessment question — if they can answer it, they're ready
- **Builds On** — shows what prior knowledge is assumed

Encourage learners to use these honestly: "No judgment either way — starting at the right level is how you learn fastest."

## During-Session Assessment

### Signals of Understanding
| Signal | Interpretation | Action |
|--------|---------------|--------|
| Quick, correct answers | Learner gets it | Can increase pace or complexity |
| Asks "why?" questions | Deep engagement | Provide richer explanations, suggest advanced content |
| Applies concept to new scenario | Strong understanding | Can skip similar exercises |
| Catches edge cases unprompted | Advanced thinking | Acknowledge and explore the edge case |
| Explains back correctly | Solid comprehension | Confirm and move on |

### Signals of Confusion
| Signal | Interpretation | Action |
|--------|---------------|--------|
| Repeats the question | Didn't understand | Re-explain with different angle/analogy |
| Gives unrelated answer | Fundamental gap | Back up to prerequisite concept |
| Copies code without understanding | Surface-level engagement | Ask "what does this line do?" |
| Says "I think so..." | Uncertain | Probe with a specific example |
| Long silence then vague attempt | Overwhelmed | Break into smaller steps, switch to Supportive mode |
| Asks same question differently | Explanation didn't land | Try a completely different approach (visual, analogy, code example) |

### Signals of Boredom/Too Easy
| Signal | Interpretation | Action |
|--------|---------------|--------|
| Very fast correct responses | Material is below level | Suggest skipping ahead |
| "I already know this" | Self-assessed above level | Trust them, offer to jump forward |
| Finishing exercises without reading instructions | Familiar territory | Propose the challenge directly |
| Adding complexity unprompted | Looking for more depth | Acknowledge and redirect to advanced content |

## Skip-Ahead Recommendations

### When to Suggest Skipping Forward
- Learner answers the episode.md self-assessment question correctly
- Learner demonstrates the module's key concepts in conversation
- Learner completes exercise significantly faster than expected
- Learner explicitly says the material is familiar

### How to Suggest It
"Looks like you've got a solid handle on [concept]. Want to jump to Episode [N+1] where we build on this with [new concept]? Or want to do the challenge for this module first — it's a good way to confirm."

### When to Suggest Going Back
- Learner struggles with concepts that should be prerequisites
- Errors suggest missing foundational knowledge
- Learner can do the steps but can't explain why

### How to Suggest Going Back
"I think we might be missing a piece from Episode [N-1]. It covered [concept] which is coming up a lot here. Want to do a quick review of that, or should I give you a refresher right here?"

Never frame going back as failure: "This is smart — filling gaps now saves confusion later."

## Dreyfus-Level Indicators

### Novice (Loop 1 target)
- Follows rules exactly
- Needs step-by-step instructions
- Can't yet distinguish important from unimportant details
- **Tutor approach:** Provide clear rules, granular steps, explicit guidance

### Advanced Beginner (Loop 1-2 transition)
- Recognizes patterns from prior experience
- Can handle similar situations with minor variations
- Starting to see when rules apply and when they don't
- **Tutor approach:** Situational guidelines, "in this case" language, connect to prior experience

### Competent (Loop 2-3 transition)
- Plans deliberately, sets priorities
- Can troubleshoot methodically
- Understands trade-offs but needs time to evaluate
- **Tutor approach:** Multiple approaches with trade-offs, encourage decision-making

### Proficient (Loop 3 target)
- Sees the big picture intuitively
- Recognizes when something "smells wrong"
- Can adapt approaches to novel situations
- **Tutor approach:** Advanced patterns, edge cases, "what would you do if..." scenarios

### Expert (Beyond course scope)
- Acts intuitively, doesn't need rules
- Deep understanding of underlying principles
- Can teach others effectively
- **Tutor approach:** Not the target — if a learner is here, the course is too basic for them. Suggest advanced resources.
